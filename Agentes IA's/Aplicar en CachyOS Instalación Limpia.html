Configuración pacman.conf

Añadir al final de /etc/pacman.conf:

sudo nano /etc/pacman.conf
[chaotic-aur]
Include = /etc/pacman.d/chaotic-mirrorlist

Instalar helper de AUR (Paru) y refrescar mirrors 

sudo pacman -S paru --needed
sudo pacman -Syyu cachyos-mirrorlist
sudo pacman -Syyu

Cambiar Kernel: Migrar a deckify-lto y actualizar flags del programador (flash y lowlatency).

⚠️ Reiniciar el PC después de actualizar el kernel.

Configuración de Firewall

Configuración UFW para Waydroid y KDE Connect

Poíticas Wifi por defecto

sudo ufw default deny incoming
sudo ufw default allow outgoing

Tráfico Web

sudo ufw allow http
sudo ufw allow https

SSH

sudo ufw allow ssh
sudo ufw allow 22/tcp

KDE Connect

sudo ufw allow 1714:1764/tcp
sudo ufw allow 1714:1764/udp

Waydroid DNS/DHCP

sudo ufw allow in on waydroid0 to any port 53 proto udp
sudo ufw allow in on waydroid0 to any port 67 proto udp

Habilitar y persistir

sudo ufw enable
sudo systemctl enable ufw

Virtualización KVM

Configurar libvirtd

sudo nano /etc/libvirt/libvirtd.conf

Descomentar:

unix_sock_group = "libvirt"
unix_sock_rw_perms = "0770"

Parámetros Kernel (Comet Lake)

Editar /etc/default/grub:

sudo nano /etc/default/grub

Buscar y reemplazar la línea:

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash intel_pstate=active intel_idle.max_cstate=2 intel_iommu=on iommu=pt i915.enable_guc=3 i915.enable_fbc=1 i915.max_vfs=7 mitigations=off nowatchdog nmi_watchdog=0 modprobe.blacklist=pcspkr"

Aplicar configuración al GRUB

sudo grub-mkconfig -o /boot/grub/grub.cfg

Permisos y Servicio

sudo usermod -aG libvirt,kvm $USER
sudo usermod -aG libvirt $(whoami)
sudo systemctl start libvirtd
sudo systemctl enable --now libvirtd
reboot

Validar después de Reiniciar

sudo virt-host-validate

PHP y Apache

Configurar PHP

sudo nano /etc/php/php.ini

Límites

  • memory_limit = 2G
  • upload_max_filesize = 256M
  • post_max_size = 256M
  • upload_max_filesize = 256M
  • post_max_size = 256M

OPcache

  • opcache.enable=1
  • opcache.memory_consumption=512
  • opcache.validate_timestamps=1
  • opcache.interned_strings_buffer=64
  • opcache.max_accelerated_files=20000
  • opcache.revalidate_freq=0

⚠️ Descomentar: pdo_mysql, bcmath, mysqli

Configurar Apache

sudo nano /etc/httpd/conf/httpd.conf

Descomentar:

LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule proxy_module modules/mod_proxy.so
LoadModule proxy_fcgi_module modules/mod_proxy_fcgi.so

Agregar:

AddHandler php-script .php
ServerName localhost
Include conf/extra/php-fpm.conf

Configurar PHP-FPM

sudo nano /etc/httpd/conf/extra/php-fpm.conf

Agregar:

<FilesMatch \.php$>
    SetHandler "proxy:unix:/run/php-fpm/php-fpm.sock|fcgi://localhost/"
</FilesMatch>

<IfModule dir_module>
    DirectoryIndex index.php index.html
</IfModule>

Iniciar Servicios

sudo systemctl start httpd php-fpm
sudo systemctl enable --now php-fpm httpd

PostgreSQL

Configuración PostgreSQL - 1° Vez

sudo -iu postgres
initdb --locale=$LANG -E UTF8 -D '/var/lib/postgres/data'

Configuración (16GB RAM)

Editar /var/lib/postgres/data/postgresql.conf: 

sudo nano /var/lib/postgres/data/postgresql.conf

Ajustar:

shared_buffers = 4GB
effective_cache_size = 8GB
maintenance_work_mem = 1GB
work_mem = 256MB
wal_buffers = 16MB

Habilitar Servicio PostgreSQL

exit
sudo systemctl enable postgresql.service --now

Creación de usuario PostgreSQL

sudo -iu postgres
psql
ALTER USER postgres WITH PASSWORD 'you_secure-password';

WordPress

Base de Datos

CREATE DATABASE wordpress;
CREATE USER 'wpuser'@'localhost' IDENTIFIED BY 'you_secure-password';
GRANT ALL PRIVILEGES ON wordpress.* TO 'wpuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Virtual Host Apache

Crear /etc/httpd/conf/extra/httpd-wordpress.conf: 

sudo nano /etc/httpd/conf/extra/httpd-wordpress.conf

Agregar:

Alias /wordpress "/usr/share/webapps/wordpress/"
<Directory "/usr/share/webapps/wordpress/">
    AllowOverride All
    Options FollowSymlinks
    Require all granted
</Directory>

Configurar Apache

sudo nano /etc/httpd/conf/httpd.conf

Agregar esta línea al final del archivo:

Include conf/extra/php-fpm.conf

Corregir Permisos de WordPress

sudo chown -R http:http /usr/share/webapps/wordpress/
sudo chmod -R 755 /usr/share/webapps/wordpress/

Finalizar instalación en: http://localhost/wordpress

Waydroid

Límites de Recursos (4GB RAM)

sudo mkdir -p /etc/systemd/system/waydroid-container.service.d/
sudo tee /etc/systemd/system/waydroid-container.service.d/override.conf << EOF
[Service]
# Aumentar memoria del contenedor de 2GB a 4GB
# (50% de RAM física es seguro)
MemoryMax=4G
MemoryHigh=3.5G

# Limitar swap (evitar I/O lento)
MemorySwapMax=512M

# Aumentar CPUs de 2 a 4 cores
AllowedCPUs=0,1,2,3

# Priorizar I/O (mejora carga de apps)
IOWeight=500
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now waydroid-container

Optimización del Sistema

Sysctl (99-master-performance.conf)

sudo nano /etc/sysctl.d/99-master-performance.conf

Memoria (16GB) & I/O SSD

# --- MEMORIA (16GB RAM) I/O & WRITEBACK (SSD Optimizado) ---
# Evita el swap prematuro y prioriza el uso de RAM física
vm.swappiness=10
# Equilibrio: No purgar caché de archivos demasiado rápido (ideal para IDEs/C++)
vm.vfs_cache_pressure=50
# Datos en RAM antes de forzar escritura al disco
vm.dirty_background_ratio=5
vm.dirty_ratio=15
vm.dirty_expire_centisecs=300
vm.dirty_writeback_centisecs=100

Red

# --- RED & CONECTIVIDAD (Dev + Waydroid/Server) ---
# Manejo de ráfagas de conexiones locales
net.core.somaxconn=1024
net.core.netdev_max_backlog=65536
net.core.default_qdisc=fq_codel
net.ipv4.tcp_fastopen=3
# Necesario para contenedores/Waydroid
net.ipv4.ip_forward=1

Scheduler

# --- SCHEDULER (CachyOS BORE / Deckify) ---
# Optimización de estados de energía y latencia de hilos
kernel.sched_cstate_aware=1
kernel.sched_child_runs_first=0
kernel.sched_tunable_scaling=1
kernel.numa_balancing=0

Recursos de Usuario

# --- RECURSOS DE USUARIO (IDEs & File Watching) ---
# Evita errores de "too many open files" en VS Code/Compilaciones
fs.inotify.max_user_watches=524288
fs.inotify.max_user_instances=512

Reiniciar Servicios

sudo sysctl --system
